## TC-EDGE-001: Неизвестная команда

**Предусловия:**
- Бот запущен

**Шаги:**
1. Отправить `/unknown_command`

**Ожидаемый результат:** Бот отвечает "Unknown command. Use /help to see available commands." Бот не падает.

**Предусловия:**
- Нет активной сессии

**Шаги:**
1. Написать "привет как дела"
2. Написать ещё один произвольный текст

**Ожидаемый результат:** Бот не отвечает ни на одно из сообщений.

## TC-EDGE-003: Изоляция проектов между чатами

**Предусловия:**
- Чат A выбрал Проект X
- Чат B использует того же бота

**Шаги:**
1. В Чате B выбрать Проект Y через `/set_project`
2. В Чате A запустить полный флоу и подтвердить задачу
3. В Чате B запустить полный флоу и подтвердить задачу

**Ожидаемый результат:** Задача из Чата A создана в Проекте X. Задача из Чата B создана в Проекте Y. Настройки не пересекаются.

---

## TC-EDGE-004: Изоляция сессий между чатами

**Предусловия:**
- Чат A имеет активную сессию

**Шаги:**
1. В Чате B отправить `/start_discussion`
2. В Чате A написать сообщение
3. В Чате B отправить `/create_task`

**Ожидаемый результат:** Новая сессия создана в Чате B. Сообщение из Чата A не попадает в Чат B. AI на шаге 3 анализирует только сообщения Чата B.

---

## TC-EDGE-005: Перезапуск бота при открытой сессии

**Предусловия:**
- Активная сессия с несколькими сообщениями
- Бот перезапускается (docker-compose restart bot)

**Шаги:**
1. Дождаться перезапуска
2. Написать сообщение в чат
3. Отправить `/create_task`

**Ожидаемый результат:** Бот снова отвечает на команды. Сообщение на шаге 2 сохраняется в существующую сессию (восстановлена из БД). На шаге 3 AI анализирует все сообщения сессии, включая те, что были до перезапуска.

---

## TC-EDGE-006: Перезапуск бота при ожидании edit-reply

**Предусловия:**
- Пользователь нажал "Редактировать", бот прислал инструкцию
- Бот перезапускается

**Шаги:**
1. Дождаться перезапуска
2. Ответить (reply) на сообщение-инструкцию
3. Отправить `/create_task`

**Ожидаемый результат:** На шаге 2 бот не обрабатывает reply как edit (in-memory editSessions очищены после перезапуска). Текст reply сохраняется в сессию как обычное сообщение. На шаге 3 показан новый preview.

**Примечание:** Известное ограничение. Тест документирует текущее поведение.

## TC-EDGE-007: Перезапуск бота при открытом preview

**Предусловия:**
- Preview с кнопками показан
- Бот перезапускается

**Шаги:**
1. Дождаться перезапуска
2. Нажать "Подтвердить" на preview

**Ожидаемый результат:** Бот обрабатывает callback. Задача может быть создана (callback handler работает). Старый preview не будет удалён ботом из чата (pendingActionMessages очищены после перезапуска).

**Примечание:** Известное ограничение. Тест документирует текущее поведение.

## TC-EDGE-008: Telegram Update без Message и CallbackQuery игнорируется

**Предусловия:**
- Бот получает update нестандартного типа (inline_query и т.п.)

**Шаги:**
1. Отправить нестандартный update боту
2. Отправить обычное сообщение боту

**Ожидаемый результат:** Бот игнорирует нестандартный update молча. На шаге 2 бот отвечает корректно.

## TC-EDGE-009: Текст кнопки с опечаткой не срабатывает как кнопка

**Предусловия:**
- Пользователь вручную набирает текст, похожий на кнопку

**Шаги:**
1. Написать "Выбрать проект" (без пробела перед текстом или другое отличие от точного текста кнопки)
2. Написать "выбрать проект" (строчными)

**Ожидаемый результат:** Бот не обрабатывает ни одно из сообщений как нажатие кнопки. Если активна сессия - сообщения сохраняются в сессию; если нет - бот молчит.

**Предусловия:**
- Сессия содержит 100+ текстовых сообщений от разных участников

**Шаги:**
1. Отправить `/create_task`

**Ожидаемый результат:** Preview показан с осмысленной задачей. Либо, если AI вернул ошибку из-за превышения лимита токенов, бот отвечает "AI суммаризация не удалась(. Попробуйте заново". Бот не падает.
