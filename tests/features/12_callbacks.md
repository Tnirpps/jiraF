# Сьют 12: Callbacks preview - Подтвердить / Редактировать / Отменить создание

---

## TC-CB-001: Владелец нажимает "Подтвердить" - задача создаётся

**Предусловия:**
- Preview показан, Todoist API доступен
- Нажимает владелец сессии

**Шаги:**
1. Нажать "Подтвердить"
2. Открыть Todoist

**Ожидаемый результат:** Бот показывает toast "Отлично! Создаю задачу." Кнопки preview исчезают. Бот присылает "Задача создана: [{заголовок}]({ссылка})". Сессия закрыта. Задача присутствует в Todoist с правильным заголовком, описанием, приоритетом, сроком и метками.

---

## TC-CB-002: Ссылка на задачу ведёт на корректный URL Todoist

**Предусловия:**
- Задача только что подтверждена (TC-CB-001)

**Шаги:**
1. Нажать на ссылку в сообщении бота

**Ожидаемый результат:** URL имеет формат `https://app.todoist.com/app/task/{id}`. Задача открывается в Todoist.

---

## TC-CB-003: Не-владелец нажимает "Подтвердить" - отказ

**Предусловия:**
- Preview показан в групповом чате
- Нажимает Пользователь B (не владелец)

**Шаги:**
1. Пользователь B нажимает "Подтвердить"

**Ожидаемый результат:** Telegram показывает toast "Только автор обсуждения может создать задачу". Кнопки preview остаются. Задача в Todoist не создана. Сессия не закрыта.

---

## TC-CB-004: Todoist недоступен при подтверждении

**Предусловия:**
- Preview показан, Todoist API недоступен

**Шаги:**
1. Владелец нажимает "Подтвердить"

**Ожидаемый результат:** Toast «Error: Failed to create task». Кнопки исчезают. Бот дополнительно присылает в чат сообщение «✅ Создание задачи отменено, продолжайте обсуждение» — баг: из-за некорректного парсинга `callbackType` в `bot.go` срабатывает ошибочный fallback вместо пустого ответа. Сессия не закрыта.

---

## TC-CB-005: Владелец нажимает "Редактировать"

**Предусловия:**
- Preview показан, нажимает владелец

**Шаги:**
1. Нажать "Редактировать"

**Ожидаемый результат:** Toast "Пожалуйста, ответьте на это сообщение с инструкциями по редактированию". Кнопки preview исчезают. Бот присылает сообщение с инструкцией и примерами. Ответ (reply) на это сообщение запустит редактирование.

---

## TC-CB-007: Владелец нажимает "Отменить создание"

**Предусловия:**
- Preview показан, нажимает владелец

**Шаги:**
1. Нажать "Отменить создание"

**Ожидаемый результат:** Toast "Создание задачи отменено". Кнопки preview исчезают. Бот присылает "Создание задачи отменено. Обсуждение продолжается." Черновик удалён из БД. Сессия остаётся открытой.

## TC-CB-009: После действия владельца кнопки preview исчезают

**Предусловия:**
- Preview показан

**Шаги:**
1. Владелец нажимает любую из трёх кнопок

**Ожидаемый результат:** Кнопки удаляются из preview-сообщения. Текст черновика в сообщении не изменился.

## TC-CB-010: Повторное нажатие на уже удалённые кнопки

**Предусловия:**
- Кнопки preview уже нажаты и удалены

**Шаги:**

**Ожидаемый результат:** Telegram не реагирует - кнопок нет в интерфейсе.
