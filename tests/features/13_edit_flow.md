# Сьют 13: Edit flow - редактирование черновика через reply

---

## TC-ED-001: Reply на edit-инструкцию - AI применяет изменения

**Предусловия:**
- Preview показан, владелец нажал "Редактировать"
- Бот прислал сообщение с инструкцией

**Шаги:**
1. Ответить (reply) на сообщение-инструкцию: "Измени заголовок на: Новое название"

**Ожидаемый результат:** Бот показывает "Задача обновлена!" и новый preview. Заголовок изменён на "Новое название". Под preview три кнопки: "Подтвердить", "Редактировать", "Отменить создание".

## TC-ED-009: Несколько итераций редактирования подряд

**Предусловия:**
- Первое редактирование выполнено (TC-ED-001), новый preview показан

**Шаги:**
1. Нажать "Редактировать" в новом preview
2. Ответить на инструкцию: "Измени приоритет на высокий"
3. Нажать "Подтвердить"

**Ожидаемый результат:** На шаге 2 появляется третий preview с "Приоритет: Высокий". На шаге 3 задача создана с финальными данными из последнего preview.

---

## TC-ED-010: Старое action-сообщение удаляется при новом редактировании

**Предусловия:**
- Было первое edit-сообщение (ожидание reply)
- Начато второе редактирование через новый preview

**Шаги:**
1. Нажать "Редактировать" в новом preview

**Ожидаемый результат:** Первое edit-сообщение удаляется из чата. В чате видно только второе сообщение с инструкцией.

## TC-ED-011: AI недоступен при редактировании

**Предусловия:**
- Ожидается reply, AI API недоступен

**Шаги:**
1. Ответить на инструкцию с правками

**Ожидаемый результат:** Бот отвечает "Error editing task". Черновик в БД не изменён. Сессия активна - можно подтвердить или отменить прежний черновик.
