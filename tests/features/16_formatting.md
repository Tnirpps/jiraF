# Сьют 16: Форматирование preview и контента задачи в Todoist

---

## TC-FMT-001: Markdown-заголовок ### в описании конвертируется в жирный текст

**Предусловия:**
- AI вернул description с "### Основные изменения"

**Шаги:**
1. Отправить `/create_task`

**Ожидаемый результат:** В preview описание содержит "*Основные изменения*" (жирный) вместо "### Основные изменения".

---

1. Отправить `/create_task`

---

## TC-FMT-005: Preview с HTTP-ссылками - веб-превью Telegram не разворачивается

**Предусловия:**
- Preview содержит "Полезные материалы" с HTTP/HTTPS ссылками

**Шаги:**
1. Получить preview-сообщение

**Ожидаемый результат:** Telegram не разворачивает веб-превью ни для одной ссылки. Сообщение компактное, без вставленных карточек.

---

## TC-FMT-006: Задача в Todoist содержит секцию "## Описание"

**Предусловия:**
- Preview показан, description не пустой

**Шаги:**
1. Нажать "Подтвердить"
2. Открыть задачу в Todoist

**Ожидаемый результат:** Описание задачи в Todoist начинается с секции "## Описание" с текстом от AI.

---

## TC-FMT-007: Задача в Todoist содержит секцию "## Детали задачи" при заполненных полях

**Предусловия:**
- Тип задачи `bug`, AI заполнил поля "Что сломано" и "Критерии проверки"

**Шаги:**
1. Нажать "Подтвердить"
2. Открыть задачу в Todoist

**Ожидаемый результат:** Описание содержит секцию "## Детали задачи" с полями в формате "- **Что сломано:** {значение}", "- **Критерии проверки:** {значение}".

---

## TC-FMT-008: Задача в Todoist содержит секцию "## Полезные материалы"

**Предусловия:**
- Preview содержит ссылки в "Полезные материалы"

**Шаги:**
1. Нажать "Подтвердить"
2. Открыть задачу в Todoist

**Ожидаемый результат:** Описание содержит секцию "## Полезные материалы" с ссылками в формате "- **{role}:** {url} — {reason}".
