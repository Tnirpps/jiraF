# Сьют 3: /set_project - выбор проекта

---

## TC-SP-001: /set_project показывает список проектов Todoist

**Предусловия:**
- Todoist API доступен
- В Todoist есть хотя бы один проект

**Шаги:**
1. Отправить `/set_project`

**Ожидаемый результат:** Бот отвечает сообщением "Выберите проект Todoist:" с inline-кнопками. Каждый проект Todoist - отдельная кнопка-строка с именем проекта.

---

## TC-SP-002: Выбор проекта из списка

**Предусловия:**
- `/set_project` отправлен, список проектов показан

**Шаги:**
1. Нажать на кнопку любого проекта

**Ожидаемый результат:** Кнопки исчезают. Бот отвечает "Проект выбран. ID: {id проекта}". Последующий `/start_discussion` запускается без запроса выбора проекта.

---

## TC-SP-003: Любой участник чата может выбрать проект

**Предусловия:**
- В групповом чате есть Пользователь A и Пользователь B
- Список проектов показан (отправлен `/set_project`)

**Шаги:**
1. Пользователь B (не владелец сессии) нажимает на кнопку проекта

**Ожидаемый результат:** Бот принимает выбор. Кнопки исчезают. Появляется "Проект выбран. ID: {id}". Проверка на владельца отсутствует.

---

## TC-SP-004: Смена уже установленного проекта

**Предусловия:**
- Проект X установлен
- В Todoist есть Проект Y

**Шаги:**
1. Отправить `/set_project`
2. Нажать на кнопку Проекта Y
3. Отправить `/start_discussion`, написать сообщение, отправить `/create_task`, подтвердить

**Ожидаемый результат:** Задача создаётся в Проекте Y. Проект X перезаписан.

---

## TC-SP-005: Смена проекта во время активной сессии

**Предусловия:**
- Активная сессия обсуждения запущена
- В Todoist есть другой проект

**Шаги:**
1. Отправить `/set_project` и выбрать другой проект
2. Написать сообщение в чат
3. Отправить `/create_task` и подтвердить

**Ожидаемый результат:** Задача создаётся в новом проекте. Сессия не прерывается при смене проекта.

---

## TC-SP-006: Todoist недоступен при вызове /set_project

**Предусловия:**
- Todoist API недоступен

**Шаги:**
1. Отправить `/set_project`

**Ожидаемый результат:** Бот отвечает сообщением, содержащим "Не удалось загрузить проекты Todoist:". Бот не падает.

---

## TC-SP-007: Todoist возвращает пустой список проектов

**Предусловия:**
- Todoist API доступен, но в аккаунте нет проектов

**Шаги:**
1. Отправить `/set_project`

**Ожидаемый результат:** Бот отвечает "В Todoist не найдено ни одного проекта." Inline-кнопки с проектами отсутствуют.

---

## TC-SP-008: Кнопка "Выбрать проект" эквивалентна команде /set_project

**Предусловия:**
- Главная клавиатура видна

**Шаги:**
1. Нажать "Выбрать проект"

**Ожидаемый результат:** Поведение идентично прямой команде `/set_project`.

---

## TC-SP-009: Разные чаты хранят независимые проекты

**Предусловия:**
- Чат A установил Проект X
- Чат B работает с тем же ботом

**Шаги:**
1. В Чате B отправить `/set_project` и выбрать Проект Y
2. В Чате A запустить полный флоу: `/start_discussion` - написать сообщение - `/create_task` - подтвердить
3. В Чате B повторить тот же флоу

**Ожидаемый результат:** Задача из Чата A создана в Проекте X. Задача из Чата B создана в Проекте Y. Настройки чатов не пересекаются.
