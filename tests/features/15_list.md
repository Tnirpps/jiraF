# Сьют 15: /list - список задач и проектов

---

## TC-LS-001: /list без аргументов - список всех задач

**Предусловия:**
- Todoist API доступен, есть задачи

**Шаги:**
1. Отправить `/list`

**Ожидаемый результат:** Бот отвечает со списком задач. Заголовок "*Ваши задачи:*". Активные задачи с символом незавершённости, завершённые - с зачёркиванием.

---

## TC-LS-002: /list tasks - список всех задач

**Предусловия:**
- Todoist API доступен

**Шаги:**
1. Отправить `/list tasks`

**Ожидаемый результат:** Ответ идентичен `/list` без аргументов.

---

## TC-LS-003: /list projects - список проектов

**Предусловия:**
- Todoist API доступен, есть проекты

**Шаги:**
1. Отправить `/list projects`

**Ожидаемый результат:** Бот отвечает со списком проектов. Заголовок "*Ваши проекты:*". Каждый проект: имя жирным, ID в обратных кавычках, подсказка `/list tasks {id}`.

---

## TC-LS-004: /list tasks {project_id} - задачи конкретного проекта

**Предусловия:**
- Todoist API доступен, известен ID проекта с задачами

**Шаги:**
1. Отправить `/list tasks {project_id}`

**Ожидаемый результат:** Бот отвечает задачами только из этого проекта. Заголовок содержит название проекта: "*Задачи в проекте {Название проекта}:*".

---

## TC-LS-005: /list {project_id} - ID первым аргументом

**Предусловия:**
- Todoist API доступен

**Шаги:**
1. Отправить `/list {project_id}`

**Ожидаемый результат:** Поведение идентично `/list tasks {project_id}`.

---

## TC-LS-006: /list tasks с несуществующим project_id

**Предусловия:**
- Todoist API доступен, project_id не существует

**Шаги:**
1. Отправить `/list tasks invalid_id_999`

**Ожидаемый результат:** Бот отвечает. Заголовок содержит ID без имени: "Задачи в проекте invalid_id_999". Список пустой или содержит сообщение "В проекте с ID invalid_id_999 задач не найдено."

---
