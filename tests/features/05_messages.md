# Сьют 5: Сохранение сообщений во время сессии

---

## TC-MSG-001: Текстовое сообщение во время сессии сохраняется

**Предусловия:**
- Активная сессия запущена

**Шаги:**
1. Написать текстовое сообщение "Нужно исправить баг с авторизацией"
2. Отправить `/create_task`

**Ожидаемый результат:** Бот молчит на шаге 1. На шаге 2 AI включает текст сообщения в анализ - preview содержит контент из этого сообщения.

---

## TC-MSG-002: Текстовое сообщение без активной сессии не сохраняется

**Предусловия:**
- Нет активной сессии

**Шаги:**
1. Написать произвольный текст
2. Отправить `/start_discussion`
3. Написать второе сообщение
4. Отправить `/create_task`

**Ожидаемый результат:** Бот молчит на шаге 1. AI на шаге 4 анализирует только второе сообщение - первое сообщение не попало в сессию.

---

## TC-MSG-003: Команда во время сессии не сохраняется как сообщение

**Предусловия:**
- Активная сессия запущена, есть хотя бы одно обычное сообщение

**Шаги:**
1. Отправить `/help`
2. Отправить `/create_task`

**Ожидаемый результат:** На шаге 1 бот выполняет команду и показывает справку. В preview на шаге 2 нет упоминания текста `/help` - команда не попала в сессию как сообщение.

---

## TC-MSG-004: Несколько участников пишут во время одной сессии

**Предусловия:**
- Групповой чат с Пользователями A, B, C
- Активная сессия запущена Пользователем A

**Шаги:**
1. Пользователь A пишет: "Надо сделать X"
2. Пользователь B пишет: "Согласен, дедлайн пятница"
3. Пользователь C пишет: "Назначить @petrov"
4. Пользователь A отправляет `/create_task`

**Ожидаемый результат:** Бот молчит на шагах 1-3. На шаге 4 AI учёл сообщения всех трёх участников - preview содержит контент из всего обсуждения.

---

## TC-MSG-005: Сообщение с URL - ссылка сохраняется

**Предусловия:**
- Активная сессия запущена

**Шаги:**
1. Написать "Посмотри логи https://logs.example.com/error-123"
2. Отправить `/create_task`

**Ожидаемый результат:** Preview содержит секцию "Полезные материалы" с `https://logs.example.com/error-123`.

---

## TC-MSG-006: Telegram text_link-сущность - URL извлекается

**Предусловия:**
- Активная сессия запущена

**Шаги:**
1. Отправить сообщение с гиперссылкой (скрытый URL под текстом, Telegram text_link entity)
2. Отправить `/create_task`

**Ожидаемый результат:** URL из гиперссылки попадает в кандидаты ссылок. AI может включить его в "Полезные материалы" preview.

---

## TC-MSG-007: Сообщение без текста (стикер, фото, файл) игнорируется

**Предусловия:**
- Активная сессия запущена, есть хотя бы одно текстовое сообщение

**Шаги:**
1. Отправить стикер
2. Отправить фото без подписи
3. Отправить `/create_task`

**Ожидаемый результат:** Бот молчит на шагах 1-2. Preview на шаге 3 не содержит упоминания стикера или фото.

---

## TC-MSG-008: Reply на чужое сообщение сохраняется как обычный текст

**Предусловия:**
- Активная сессия запущена
- В чате есть сообщение от другого участника

**Шаги:**
1. Ответить (reply) на сообщение другого участника с текстом
2. Отправить `/create_task`

**Ожидаемый результат:** Бот молчит на шаге 1. Текст reply-сообщения попал в сессию и учтён AI на шаге 2.

---

## TC-MSG-009: Reply на edit-инструкцию бота обрабатывается как редактирование

**Предусловия:**
- Нажата кнопка "Редактировать" в preview
- Бот прислал сообщение с инструкцией

**Шаги:**
1. Ответить (reply) на сообщение-инструкцию: "Установи приоритет высокий"

**Ожидаемый результат:** Бот обращается к AI и показывает обновлённый preview. Это сообщение не сохраняется в список сообщений сессии.

---

## TC-MSG-010: Обычное сообщение при ожидании edit-reply не трактуется как редактирование

**Предусловия:**
- Нажата кнопка "Редактировать", бот ждёт reply

**Шаги:**
1. Написать сообщение без reply (просто текст в чат, не ответ на сообщение-инструкцию)

**Ожидаемый результат:** Бот молчит. Preview не изменился. Сообщение сохраняется в сессию и учитывается при следующем `/create_task`.
