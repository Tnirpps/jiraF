# Сьют 14: /cancel - завершение обсуждения без задачи

---

## TC-CL-001: /cancel без активной сессии

**Предусловия:**
- Нет активной сессии

**Шаги:**
1. Отправить `/cancel`

**Ожидаемый результат:** Бот отвечает "Нет активного обсуждения."

---

## TC-CL-002: /cancel владельцем - диалог подтверждения

**Предусловия:**
- Активная сессия запущена владельцем

**Шаги:**
1. Отправить `/cancel`

**Ожидаемый результат:** Бот отвечает "Завершить обсуждение без создания задачи?" с двумя inline-кнопками: "Завершить" и "Продолжить".

---

## TC-CL-003: Не-владелец вызывает /cancel

**Предусловия:**
- Групповой чат, Пользователь A запустил сессию

**Шаги:**
1. Пользователь B отправляет `/cancel`

**Ожидаемый результат:** Бот отвечает "Только автор обсуждения может завершить его." Сессия не закрыта.

---

## TC-CL-004: Нажать "Завершить" - сессия закрывается

**Предусловия:**
- Диалог подтверждения показан (TC-CL-002 выполнен)

**Шаги:**
1. Владелец нажимает "Завершить"
2. Написать сообщение в чат
3. Отправить `/start_discussion`

**Ожидаемый результат:** Кнопки исчезают. Бот отвечает "Обсуждение завершено без создания задачи." Сообщение на шаге 2 не сохраняется в сессию. На шаге 3 новая сессия создаётся без ошибки.

---

## TC-CL-005: Нажать "Продолжить" - сессия остаётся открытой

**Предусловия:**
- Диалог подтверждения показан

**Шаги:**
1. Владелец нажимает "Продолжить"
2. Написать сообщение
3. Отправить `/create_task`

**Ожидаемый результат:** Кнопки исчезают. Бот отвечает "Обсуждение продолжается." Сообщение на шаге 2 сохраняется в сессию. На шаге 3 preview показан.

---

## TC-CL-006: Не-владелец нажимает "Завершить" - отказ

**Предусловия:**
- Диалог подтверждения показан в групповом чате

**Шаги:**
1. Пользователь B нажимает "Завершить"

**Ожидаемый результат:** Toast "Только автор обсуждения может завершить его". Кнопки остаются. Сессия не закрыта.

---

## TC-CL-007: Не-владелец нажимает "Продолжить" - отказ

**Предусловия:**
- Диалог подтверждения показан в групповом чате

**Шаги:**
1. Пользователь B нажимает "Продолжить"

**Ожидаемый результат:** Toast "Только автор обсуждения может продолжить обсуждение". Кнопки остаются.

---

## TC-CL-008: /cancel при активном preview задачи

**Предусловия:**
- Preview с кнопками показан
- Пользователь не нажал ни одну кнопку preview

**Шаги:**
1. Владелец отправляет `/cancel`
2. Владелец нажимает "Завершить"

**Ожидаемый результат:** На шаге 1 показан диалог подтверждения. На шаге 2 сессия закрывается. Кнопки preview в старом сообщении остаются (бот не удаляет их при /cancel).

---

## TC-CL-009: Кнопка "Завершить обсуждение" эквивалентна /cancel

**Предусловия:**
- Активная сессия, главная клавиатура видна

**Шаги:**
1. Нажать "Завершить обсуждение"

**Ожидаемый результат:** Поведение идентично прямой команде `/cancel`. Показан диалог с кнопками "Завершить" и "Продолжить".
