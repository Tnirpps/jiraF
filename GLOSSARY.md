# Глоссарий jiraF

## Основные термины

| Термин | Определение |
|--------|-------------|
| **Сессия** | Активный период обсуждения в чате. Начинается командой `/start_discussion`, заканчивается `/cancel` или созданием задачи. Одна сессия на чат. |
| **Черновик (Draft)** | Промежуточное представление задачи, сформированное AI из сообщений сессии. Содержит заголовок, описание, срок, приоритет. |
| **Long Polling** | Метод получения обновлений от Telegram API. Бот опрашивает сервер каждые 60 секунд. Альтернатива — webhook. |
| **Privacy Mode** | Настройка бота в @BotFather. При OFF бот видит все сообщения в групповых чатах (не только команды). |
| **Assignee Note** | Текстовая заметка о предполагаемом исполнителе. Извлекается из @упоминаний и фраз типа "назначь на Ивана". |
| **Callback** | Нажатие на inline-кнопку (Подтвердить/Редактировать/Отменить). Обрабатывается через callback_query. |
| **AI Provider** | Внешний сервис для суммаризации и редактирования черновиков. Поддерживаются: YandexGPT, OpenRouter. |
| **Идемпотентность** | Свойство операций: повторное нажатие кнопки не создаст дубль задачи. Достигается проверкой по `session_id`. |
| **Owner ID** | ID пользователя, запустившего сессию. Только владелец может создать задачу по итогам обсуждения. |

## Сокращения

| Сокращение | Расшифровка |
|------------|-------------|
| **AI** | Artificial Intelligence — искусственный интеллект |
| **API** | Application Programming Interface — интерфейс программирования приложений |
| **DB / БД** | Database — база данных |
| **MVP** | Minimum Viable Product — минимально жизнеспособный продукт |
| **ADR** | Architecture Decision Record — документ об архитектурном решении |
| **SRE** | Site Reliability Engineering — инженерия надёжности |
| **TZ** | Time Zone — часовой пояс (Europe/Moscow) |
| **UTC** | Coordinated Universal Time — всемирное координированное время |
| **MSK** | Moscow Standard Time — московское время (UTC+3) |
| **ISO** | International Organization for Standardization — международный стандарт (дат) |
| **JSON** | JavaScript Object Notation — формат обмена данными |
| **REST** | Representational State Transfer — архитектурный стиль API |
| **CI/CD** | Continuous Integration / Continuous Delivery — непрерывная интеграция и доставка |
